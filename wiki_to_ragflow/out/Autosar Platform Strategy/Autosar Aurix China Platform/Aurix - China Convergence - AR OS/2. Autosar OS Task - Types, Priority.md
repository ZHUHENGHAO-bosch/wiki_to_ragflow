## Tasks

Two different task concepts are provided by the OSEK operating system

- basic tasks
- extended tasks

### Basic Tasks

Basic Tasks have only three states as shown below

![](../../../_images/2.%20Autosar%20OS%20Task%20-%20Types%2C%20Priority/image-2023-5-6_13-50-0.png)

Basic tasks only release the processor if

- they terminate
- operating system switches to a higher-priority task (If the task is configured as Preemptable)
- an interrupt occurs which causes the processor to switch to an interrupt service routine (ISR)

### Extended Tasks

Extended tasks are distinguished from basic tasks by being allowed to use the operating system call WaitEvent, which may result in a waiting state

![](../../../_images/2.%20Autosar%20OS%20Task%20-%20Types%2C%20Priority/image-2023-5-6_13-58-9.png)

### Task priority

The scheduler decides on the basis of the task priority which is the next of the ready tasks to be transferred into the running state

> **INFO: Dynamic Priority**
> To enhance efficiency, a dynamic priority management is not supported. Accordingly the priority of a task is defined statically, i.e. the user cannot change it at the time of execution. However, in particular cases the operating system can treat a task with a defined higher priority (Priority Ceiling Protocol)

Tasks on the same priority level are started depending on their order of activation,

A preempted task is considered to be the first (oldest) task in the ready list of its current priority

![](../../../_images/2.%20Autosar%20OS%20Task%20-%20Types%2C%20Priority/image-2023-5-6_14-20-38.png)

The following fundamental steps are necessary to determine the next task to be processed:

- Step 1: The scheduler searches for all tasks in the ready/running state.
- Step 2: From the set of tasks in the ready/running state, the scheduler determines the set of tasks with the highest priority.
- Step 3: Within the set of tasks in the ready/running state and of highest priority, the scheduler finds the oldest task.
