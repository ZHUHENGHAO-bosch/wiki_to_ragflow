### Introduction

Integration testing involves combining units into modules and testing those modules or groups of modules. The aim of integration testing is to expose deficiencies as units are tested in combination.

What is Unit testing? Unit test objective is to exercise all paths through the code base using structural coverage analysis. (GTest)

#### Level-1 Integration Testing

All existing integration tests shall be executed to verify that public branch (mainline) will not break due to changes done by SW Project Contributor on private branch

![](../../../../_images/Aurix%20China%20Convergence%20Platform%20-%20Integration%20testing/image-2023-3-31_9-21-35.png)

#### Level-2 Integration Testing

The SW Level-2 Integration activity contains several steps performed during the construction of SW, starting with the synchronization with the defined SW branch and continuing with its verification using the methods unit testing and static code analysis supplemented by integration tests. After successfully performing all automated tests this integration step ends with delivering a release candidate.
