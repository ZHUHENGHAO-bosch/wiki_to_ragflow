Step 2: Create a .robot file with the module name.

Example, UTFTestLCM.robot for LCM

> **INFO: Robot framework Syntax**
> Check this link for Robot file syntax Robot Framework User Guide

Step 3: Create an Test Suite. In our example, we are creating a file for LCM called UTFTestLCM.robot which is an Test Suite

![](../../../../_images/8.%20Writing%20Test%20case%20using%20Robot%20file/image-2023-3-28_14-2-45.png)

Test Suite will have different sections

- Settings - Importing test library, documenation, resource file and variable file
- Keywords - creating keywords from existing lower-level keywords
- Variables - Defining variables that can be used elsewhere in the test data.
- Test Cases - Creating test case from available keywords.
- Comments - Additional comments or data. Ignored by Robot Framework.

First we need write the settings of a test suite.

Settings

- Documentation - Proper name of the Test suite
- Suite Setup - Initialization for the test suite → Inside initialize launh the dlt connector as shown in example
- Test setup is something that is executed before a test case, and a Test teardown is executed after a test case

![](../../../../_images/8.%20Writing%20Test%20case%20using%20Robot%20file/image-2023-3-28_15-0-4.png)

Step 4: Write test Cases inside test suite

Test cases shall be written inside *** Test Cases *** Section.

In our example, we have two test cases 1. Test PSC Shutdown 2. Test PSC SOH
