### Steps for initial build

Step 1: Copy the folder UTFTestModuleName from the path Autosar\clusterbase\integration-test\Service.UTFManager and place it inside Config\Autosar\integration-test\Service.UTFManager.Aurix

Step 2: Rename ModuleName with the actual module name (in this example we are taking LCM as module name)  and remove _template from file name

![](../../../../_images/2.%20Build%20UTF%20Test%20case%20Example/image-2023-3-25_15-0-37.png)

Step 3: Cmake changes

- Change the actual module name in LINKER_FLAGS

![](../../../../_images/2.%20Build%20UTF%20Test%20case%20Example/image-2023-3-25_15-23-53.png)

- Change the module name in inFiles section
- Change the module name in cmt_pf_module_newConfig option
- add the required libraries using cmt_pf_module_addLibraries
- We need to add -no_data_delete option in cmake (This is needed because of special linking in UTM mdoule) target_compile_options(UTFTestLCM PRIVATE -no_data_delete)

Step 4: Add our test case folder in Service.UTFManager.Aurix cmake by using add_subdirectory

Step 5: Add our module in FeatureListAurix.txt which is located in this path Config\Autosar\integration-test\Service.UTFManager.Aurix

The Order we define in this list will be the Module ID which will be used in injectionm command

For example, Module ID for our example is 1

![](../../../../_images/2.%20Build%20UTF%20Test%20case%20Example/image-2023-3-25_16-10-35.png)

Step 6: Add our module in Main Cmake file (CMakeLists.cmake)

It is important to add INTTEST so that the folder will be included in UTF binary creation.

cmt_use_pf_features(UTFTestXXX    INCLUDEPATH LIBRARY INTTEST)

![](../../../../_images/2.%20Build%20UTF%20Test%20case%20Example/image-2023-3-25_16-14-6.png)

Step 7: Change Modulename inside each file that we copied in Step 1

Do this change in all files

![](../../../../_images/2.%20Build%20UTF%20Test%20case%20Example/image-2023-3-25_16-18-40.png)

Step 8: Now give normal build (Build.bat xxx)

You will find a seperate binary for our module inside software packets folder. Below is the .out for our example

![](../../../../_images/2.%20Build%20UTF%20Test%20case%20Example/image-2023-3-25_16-22-49.png)

> **INFO: Flashing**
> For each Module a separate binary will be generated, and this binary shall be used for integration test cases for that module.
