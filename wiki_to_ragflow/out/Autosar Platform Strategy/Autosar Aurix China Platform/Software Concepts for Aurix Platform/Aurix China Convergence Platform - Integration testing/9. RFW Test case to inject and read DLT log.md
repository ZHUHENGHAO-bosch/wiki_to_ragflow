# 9. RFW Test case to inject and read DLT log

Below is the example for How to write test case for injecting a dlt command and reading the delt logs in RFW

Step1: Make sure the TestBench.py has correct configuration .

Path: Config\Autosar\integration-test\Service.UTFManager.Aurix\RFW_UTF\Configuration

Make sure COM port is as per the device manager port

![](../../../../_images/9.%20RFW%20Test%20case%20to%20inject%20and%20read%20DLT%20log/image-2023-3-29_16-12-35.png)

Step 2: In Test Suite Initialize use the key word launch dlt connector to connect to launch connection to DLT daemon .

![](../../../../_images/9.%20RFW%20Test%20case%20to%20inject%20and%20read%20DLT%20log/image-2023-3-29_16-16-37.png)

Step 3: To inject we need to use injection key word. Explained the format in below picture

![](../../../../_images/9.%20RFW%20Test%20case%20to%20inject%20and%20read%20DLT%20log/image-2023-3-29_16-37-42.png)

We can use searchstring and timeout in injection command to check the expected dlt log

Step 4: Monitoring and Searching for a particular Dlt log

we need to use two key words, one to start monitor and another to search the logs

- Start Monitoring
- Search Logs <search_string> <timeout>

![](../../../../_images/9.%20RFW%20Test%20case%20to%20inject%20and%20read%20DLT%20log/image-2023-3-29_16-47-42.png)

Step 5: The output.xml in this path Config\Autosar\integration-test\Service.UTFManager.Aurix\RFW_UTF\_Reports will have complete details on DLT logs

![](../../../../_images/9.%20RFW%20Test%20case%20to%20inject%20and%20read%20DLT%20log/image-2023-3-29_17-37-26.png)

If the search string is found, the test case will pass, otherwise it will fail
