## 2. Document structure and functiona lity

The relevant files of resViz mainly include _install_resviz.bat, _run_resviz.bat, memcfg_TRICORE.json,project_config.json,log_chery_codev_app.log, ram_rom_report_chery_xxx.xlsx , and the input map file (e.g., GM_VCUPLUS.map)

| File Name | Type | Description |
| --- | --- | --- |
| _install_resviz.bat | Batch File | Automates the deployment of the resViz environment, including installing dependencies and copying configuration templates. |
| _run_resviz.bat | Batch File | Configures the runtime environment (sets the Python path in the system environment variables) and invokes the resviz tool with command-line parameters. |
| GM_VCUPLUS.map | Map File | The map file generated by the GHS toolchain, containing memory information of sections and symbols, serving as the core input for resViz. |
| memcfg_TRICORE.json | JSON File | Specifies the architecture, defines information for multiple memory blocks, distinguishes between FLASH and RAM types, and enables resViz to identify memory attributes and boundaries. |
| project_config.json | JSON File | Configures project-level memory budgets: available ROM/RAM, global limits, per-domain budgets (linked to libs/objs), and checks to ignore. |
| ram_rom_report_xxx.xlsx | Excel Worksheet | Outputs the final memory analysis results. |
| log_GM_VCUPLUS.log | Text File | Records the execution process of resViz, used for debugging parsing errors or over-budget issues. |

### 2.1. Functions of each document

(1) _install_resviz.bat:

On Windows, it is recommended to use the TCC tool and include resViz as part of the ToolCollection, leveraging the new modular Python approach to manage the tool environment.

The _install_resviz.bat batch file is an automated configuration script based on this solution. It first configures Python environment variables, then checks if resViz is installed. If not, it automatically pulls and installs the specified version rb-resviz==6.2.2 from the Bosch internal repository, supporting version specification and enabling rapid environment configuration.

More info: For further details or configuration methods in the Linux environment, please refer to link resViz - XC ADAS Wiki - Docupedia .

![](../../../../_images/Taiji%20-%20RAMROM%20summary%20using%20resviz/image-2025-10-24_9-10-15.png)

(2) _run_resviz.bat：

_run_resviz.bat is a batch file for launching the resViz tool, with core functions of configuring the runtime environment and automating resViz invocation for memory analysis:

- It configures the environment via set commands: specifying the path to GM_VCUPLUS.map as MAP_PATH and setting the system path for the Python runtime environment (PATH);
- It invokes the resViz tool with parameters including the map file, project configuration (project_config.json), controller architecture (TRICORE), and device memory description (memcfg_TRICORE.json), while enabling logging and options to ignore map parsing errors for automated memory analysis execution.

![](../../../../_images/Taiji%20-%20RAMROM%20summary%20using%20resviz/image-2025-10-24_9-30-48.png)

(3) GM_VCUPLUS.map：

A map file generated by the GHS linker. When generating it, follow these linker option rules: Parameters -Mn, -ML, and -Ma must not be used. When mapfile_type is 1 or 2, -Mu must be specified; when mapfile_type is 3, -Mu must not be specified. This file contains critical information such as symbols, which resViz depends on to analyze memory resources.

Note： If the map file is generated with incorrect parameters, resViz will not function properly！！！

More info: Prerequisites — resViz documentation

(4)memcfg_TRICORE.json：

The JSON structure of this file consists of two key parts, with the meaning of each field as follows:

- "Architecture" field:

It specifies the architecture type of the target hardware as "TRICORE", allowing the resViz tool to adapt to the memory management rules of this architecture.

- "Memory Blocks" array:

This array contains configuration objects for multiple memory blocks. Each object defines an independent memory region, and the fields are explained as follows:

"Name": The identifier of the memory block (e.g., "PFL01", "PFL11"), used to distinguish different memory regions.

"Address": The start address of the memory block (e.g., "0x80F00000"), defining the starting position of the memory region in the address space.

"Size": The capacity of the memory block (e.g., "0x210000"), indicating the length of the usable memory space in this region.

"Type": The type of the memory block (e.g., "FLASH"), indicating that the region belongs to the flash memory type (to distinguish it from other memory types such as RAM and ROM).

![](../../../../_images/Taiji%20-%20RAMROM%20summary%20using%20resviz/image-2025-10-24_12-18-39.png)

(5) project_config.json：

This JSON template configures memory budgets (RAM/ROM) for Aurix TC3XXX, defining available memory, global limits, domain-specific allocations, and ignored checks.

- Key Sections ：

”Available“: Specifies total available RAM/ROM address spaces for Aurix TC3XXX (in decimal).

“Global”: Defines global RAM/ROM budgets (to reserve free space) and includes sub-entries:

- NODOMAIN: Budget for uncategorized items.

- LINKER: Budget for linker symbols.

“Domains”: Array of domain-specific budgets, each with:

- name: Domain identifier.

- rom/ram: Allocated memory (decimal).

- libs/objs: Associated libraries/object files.

“Ignore_Checks”: List of checks to skip (e.g., global budget vs. available memory).

“Ignored_SCOM_Symbols”: SCOM symbols excluded from checks (non-runnable/mempool/port instances).

![](../../../../_images/Taiji%20-%20RAMROM%20summary%20using%20resviz/image-2025-10-24_12-32-2.png)

(6) ram_rom_report_xxx.xlsx：

ram_rom_report_xxx.xlsx is a memory analysis report file that presents detailed data on RAM/ROM budgets, usage, quotas, and free space for each domain and the global scope under the Aurix TC3XXX architecture.

It consists of multiple worksheets (including Domains, Symbols, Libraries, Objects, and Memory Use) , providing categorized statistics on memory resources. This report serves as an intuitive basis for memory resource monitoring, compliance checking, and optimization.

![](../../../../_images/Taiji%20-%20RAMROM%20summary%20using%20resviz/image-2025-10-24_12-58-37.png)

(7) log_GM_VCUPLUS.log：

Records key information during resViz analysis, including:

- 1. Unused domain mappings (listing functional domains and their associated unused library files);
- 2. List of sections reducing available memory resources (e.g., section .heap occupies 1024 bytes of RAM).

This log helps troubleshoot configuration redundancy and memory usage anomalies.

![](../../../../_images/Taiji%20-%20RAMROM%20summary%20using%20resviz/image-2025-10-24_13-16-24.png)

## 3. How to use

### 3.1. step 1

Please download through the link RAMROM_resviz.zip to obtain related files .

![](../../../../_images/Taiji%20-%20RAMROM%20summary%20using%20resviz/image-2025-10-24_13-23-59.png)

### 3.2. step 2

After running _install_resviz.bat , this output indicates that resViz has already been successfully installed on the system, so no further installation is required.

![](../../../../_images/Taiji%20-%20RAMROM%20summary%20using%20resviz/image-2025-10-24_13-26-11.png)

### 3.3. step 3

In CMakeLists_APP.cmake (e.g., located at .\Build\CmakeLists ), adjust the LINK_FLAGS to configure GHS linker parameters.

Locate and modify LINK_FLAGS using set_target_properties in the relevant CMake configuration file, and add parameters such as -Mn, -Ml, -Mapfile_type=2, -Mu , while specifying the map file output path.

Note： These settings ensure the generated map file is formatted correctly, allowing resViz to parse symbols and memory sections properly.

![](../../../../_images/Taiji%20-%20RAMROM%20summary%20using%20resviz/image-2025-10-24_13-33-1.png)

### 3.4. step 4

Configure project_config.json and memcfg_TRICORE.json according to the project, place them in the same directory as _run_resviz.bat, and ensure the map file path is correct.

![](../../../../_images/Taiji%20-%20RAMROM%20summary%20using%20resviz/image-2025-10-24_13-55-59.png)

### 3.5. step 5

Run _run_resviz.bat , and you can view the spreadsheet and log data in the same directory.
