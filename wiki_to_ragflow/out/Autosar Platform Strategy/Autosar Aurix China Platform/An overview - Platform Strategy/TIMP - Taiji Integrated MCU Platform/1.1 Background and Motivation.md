## Overview

Currently in XC-CN projects, we have multiple projects and platform running with different architecture and with different setups, which makes it difficult for team to work in different projects at same time.

Our motivation is to create an integrated platform for all required solutions, which makes it easier for future and current development.

We will have more details on Motivation and pain points in following sections

### Pain points

We have multiple platforms such as Taiji for IDC, W3 platform, J6E platform and GM/PATAC based platform

Each platform has its own different modules and concepts, like:

- Completely different build environment and repo structures
- Services implemented in each platform in different way but for same purpose. Example: RTMO, ADC controller, Fault handling, logging, LCM etc.,
- Documentation done in own platform area -> cannot be reused directly in other platforms
- Different tools – Example: Some platform use Goggle test and some use CPP test for UT

With these differences, developers find difficult to work across different projects and for each project there is lot of effort wasted in setting the proper environment
