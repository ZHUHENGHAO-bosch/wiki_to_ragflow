## Overview

This page is to understand the Orin Y Power sequence and the relevant code

The orin_pwrmgr module is a component that runs on the AURIX TC3xx MCU and is responsible for managing the power lifecycle of an NVIDIA Orin SoC.

In plain terms: the MCU acts as a "supervisor" for the Orin SoC -- it controls power rails, monitors voltages and temperatures, handles safe boot chain selection, manages IST (In-System Test) execution, and processes critical failure reports from the SoC via SPI/Etherne

### Architecture Overview

The module is composed of 4 major composites plus supporting layers:

|   |   |
| --- | --- |
| MCU_RefSwModules | Reference (reusable) SW: Platform Power Manager orchestration, I2C driver, Safety Monitor HW abstraction, IST Manager interfaces, Communication coordinator, Common interface |
| NvMCU_SwModules | NVIDIA-MCU domain SW: Orin power control (VRS-10/11 regulators), voltage monitoring (VMON), temperature monitoring (TMON), IST Manager core, boot chain configuration, FOH (Fail Operational Handler), config/error codes |
| MCU_PlaceHolder | Placeholder/adapter SW: Centralized error handler, Chain-C processing, debug shell |
| MCU_NvStubs | NV stubs for NVM Info and Shell interfaces |
| rte | Custom RTE layer bridging all composites via FIFO-based error notification, power request/state signaling, IST status sharing, UDP data exchange |

### OrinY Power sequence

The power-up is driven by the pPltfPwrMgr_PwrUpOrinStJumpTable in SW_PltfPwrMgr.c . It consists of 14 sequential states executed cyclically from MCU_PltfPwrMgr_SW_RE_Cyclic

![](../../_images/4.0%20Taiji%20AI%20Box%20-%20Orin%20Y%20Power%20Management/image-2026-4-29_13-48-16.png)

#### Step 0: MCU_PLTFPWRMGR_PWRUP_ORIN_TRIG -- Pre-PowerUp

lOrinPwrCtrl_PrePwrUp

| Pin | Direction | Action | Purpose |
| --- | --- | --- | --- |
| SOC_RESET | OUT | Set HIGH | Hold Orin in reset (active-low reset, inverted on P3710+) |
| SOC_IST_REQ | OUT | Set LOW | De-assert IST request |
| SOC_WAKE | OUT | Set LOW | De-assert wake signal |
| SOC_SEQ_VMON_EN | OUT | Set LOW | Disable VRS10 sequencer/VMON |
| SOC_SEQ_VMON_SLP | OUT | Set HIGH | Put VRS10 in sleep mode |
| SOC_FORCE_RECOVERY | OUT | Set HIGH | De-assert force recovery (active-low) |

#### Step 1: MCU_PLTFPWRMGR_ENA_EXT_PWR -- Enable Pre-Regulator

lPltfPwrMgr_EnablePREREG

| Pin | Direction | Action | Purpose |
| --- | --- | --- | --- |
| VRS11_U5V0S_ON | OUT | Set HIGH | Enable 5V switched supply rail |
| (delay 500us) | -- | wait | Settling time |
| U15V5S_PRE_ON | OUT | Set HIGH | Enable 15.5V pre-regulator |

Wait: Polls for Power-Good (PG) with MCU_PLTFPWRMGR_PREREG_PG_WAIT_TIME time

#### Step 2: MCU_PLTFPWRMGR_PWRUP_ORIN_CONDTION_CHECK -- Pre-condition Check

lPltfPwrMgr_Hndl_PwrUpOrinCondCheck

| Pin | Direction | Action | Purpose |
| --- | --- | --- | --- |
| (ADC check) | IN | Read | Check pre-regulator voltage levels are within range |

Wait: Debounced check with MCU_PLTFPWRMGR_ORIN_POWERUP_COND_PASS_DEB_TIME using Time_GetEcuTimeMs timestamps. Also has a timeout MCU_PLTFPWRMGR_ORIN_POWERUP_CHECK_TIMEOUT

#### Step 3: MCU_PLTFPWRMGR_TRIG_IST -- Trigger IST

#### lPltfPwrMgr_Hndl_TrigIST

#### Configures IST Manager for this boot. No direct GPIO -- sets IST parameters via RTE.

#### Step 4: MCU_PLTFPWRMGR_INIT_TMON -- Initialize Temperature Monitor

lPltfPwrMgr_Hndl_InitTMON

Initializes the Orin TMON sensor over I2C (TMP451). No direct GPIO pin toggle.

#### Step 5: MCU_PLTFPWRMGR_INIT_VMON -- Initialize Voltage Monitor

lPltfPwrMgr_Hndl_InitVMON

Initializes voltage monitoring (VRS10/VRS11) over I2C. No direct GPIO toggle

#### Step 6: MCU_PLTFPWRMGR_PROGRAM_VMON -- Program VMON Thresholds

lPltfPwrMgr_Hndl_ProgramVMON

Programs OV/UV thresholds on VRS10/VRS11 PMIC via I2C. No direct GPIO toggle.

#### Step 7: MCU_PLTFPWRMGR_PWRUP_VRS10 -- Power-Up VRS10 (Main Orin Supply)

lPltfPwrMgr_Hndl_PwrUpVRS10

| Pin | Direction | Action | Purpose |
| --- | --- | --- | --- |
| VRS10_VRS11_NIRQ | IN | Read - expect HIGH | Verify no pre-existing faults on VMON |
| (I2C) | -- | VRS10_Init, VRS11_Init | Configure VRS10/VRS11 PMIC registers |
| (I2C) | -- | VRS10_Enable | Enable VRS10 power output |

Wait (RUNNING phase): Polls for NVMCU_ORINPWRCTRL_VRS10_PG_WAIT_TIME , then verifies

| Pin / Signal | Direction | Expected | Purpose |
| --- | --- | --- | --- |
| VRS10_VRS11_NIRQ | IN | HIGH | No fault interrupt from PMIC |
| LCVR_PG_ADC | IN (ADC) | HIGH (voltage OK) | LCVR Power-Good |
| LCVR_FSI_PG_ADC | IN (ADC) | HIGH (voltage OK) | FSI Power-Good |
| LCVR_AO_PG_ADC | IN (ADC) | HIGH (voltage OK) | AO Power-Good |

On success, enables FUNC_NIRQ continuous monitoring

#### Step 8: MCU_PLTFPWRMGR_VERIFY_PWRUP_VMON -- Verify Power-Up Sequence

lPltfPwrMgr_Hndl_VerifyPwrUpSeq

Verifies all VMON rails are stable after VRS10 power-up. Reads voltage monitoring status.

#### Step 9: MCU_PLTFPWRMGR_XFI_SOPT -- XFI SOPT Test

Handler: lPltfPwrMgr_Hndl_XfiSopt (line 2865)

Currently bypassed ( PWRSEQ_BYPASS ). Immediately advances.

#### Step 10: MCU_PLTFPWRMGR_BOOTCHAIN_SEL -- Boot Chain Selection

lPltfPwrMgr_Hndl_BootChain_Sel

| Pin | Direction | Action | Purpose |
| --- | --- | --- | --- |
| SOC_BOOTCHAIN_0 | OUT | HIGH or LOW | Boot chain bit 0 (selects A/B/C/D chain) |
| SOC_BOOTCHAIN_1 | OUT | HIGH or LOW | Boot chain bit 1 |
| SOC_IST_REQ | OUT | HIGH (if IST boot) | Signal IST boot to Orin |

#### Step 11: MCU_PLTFPWRMGR_SETPRERSTCOND -- Set Pre-Reset Conditions

Handler: lPltfPwrMgr_Hndl_SetPreResetCond Delegates to: NvMCU_OrinPwrCtrl via NVMCU_ORINPWRCTRL_REQ_SETPRERESETCOND Impl: lOrinPwrCtrl_SetPreRstCond

| Pin | Direction | Action | Purpose |
| --- | --- | --- | --- |
| SOC_FORCE_RECOVERY | OUT | HIGH or LOW | Set recovery mode if requested (active-low) |
| SOC_XFI_REFCLK_EN | OUT | Set HIGH | Enable XFI Ethernet reference clock to Orin |
| (I2C) | -- | VRS10_ClearFrcAct | Clear VRS10 Force-Active bit (allow sleep transitions) |

#### Step 12: MCU_PLTFPWRMGR_RLS_RST_TEGRA -- Release Orin Reset

Handler: lPltfPwrMgr_Hndl_RlsRstTegra Delegates to: NvMCU_OrinPwrCtrl via NVMCU_ORINPWRCTRL_REQ_REL_RESET Impl: lOrinPwrCtrl_RlsRst

| Pin | Direction | Action | Purpose |
| --- | --- | --- | --- |
| SOC_RESET | OUT | Set LOW | Release Orin from reset (active-low, inverted) -- Orin starts booting |

Post-reset actions (same handler):

- Enables Orin TMON thermal monitoring
- Enables VRS11 PG monitoring (if not in recovery mode)
- Starts FOH (First Orin Heartbeat) monitoring with delay

#### Step 13: MCU_PLTFPWRMGR_PWRUP_ORIN_SEQ_COMPLETE -- Sequence Complete

Handler: lPltfPwrMgr_Hndl_PwrUpOrinSeqEnd (line 3793)

Transitions state to MCU_PLTFPWRMGR_PWRUP_ORIN_FINISH . Orin is now booting.
