A.Use the folder in the repo directly - recommend

After executing building，all SAIL flashing related files will be automaticly updated in the path: \cust_apl\build\Qcimages\Prog_Sail

Simply use this folder in Step 2 to flash the SAIL

![](../../../_images/2.0%20SAIL%20-%20How%20to%20Flash%20and%20debug%20SAIL/image-2025-8-15_10-49-44.png)

B.Copy elf file manually - deprecated

Copy the signed elf file from build repo and paste to build images with name as sailsw1.elf

make sure you copy the signed elf. If you copy the normal elf, hypervisor will not load the El1 elf into SRAM.

![](../../../_images/2.0%20SAIL%20-%20How%20to%20Flash%20and%20debug%20SAIL/image-2024-10-16_20-30-17.png)

Paste in below folder where we downloaded SOC binary

![](../../../_images/2.0%20SAIL%20-%20How%20to%20Flash%20and%20debug%20SAIL/image-2024-10-16_20-31-10.png)

Copy the Hypervisor elf from repo and paste to Sail image folder

![](../../../_images/2.0%20SAIL%20-%20How%20to%20Flash%20and%20debug%20SAIL/image-2024-10-10_16-56-27.png)

If the hypervisor image is not flashed with the one in repo, the code will not jump to EL1 SW

#### 1.3. B. Load the elf in t32 manually - deprecated

Step 1: Open t32start.exe from any of the trace32 installation folder - from antifactory or from J6E repo

Select Power view instance as Arm and click on start

![](../../../_images/2.0%20SAIL%20-%20How%20to%20Flash%20and%20debug%20SAIL/image-2024-9-14_11-7-15.png)

Now the T32 Powerview window will be open as shown below

![](../../../_images/2.0%20SAIL%20-%20How%20to%20Flash%20and%20debug%20SAIL/image-2024-9-14_11-8-16.png)

Step 2: Configure DAP settings

In T32 Power window, run the below attached CMM script → this will configure required DAP settings

SAIL_Connect_XC.cmm

![](../../../_images/2.0%20SAIL%20-%20How%20to%20Flash%20and%20debug%20SAIL/image-2024-9-14_11-9-56.png)

Click Run script and choose the downloaded file

No Errors should be there and there will be a Cortex R52 option in menu bar

![](../../../_images/2.0%20SAIL%20-%20How%20to%20Flash%20and%20debug%20SAIL/image-2024-9-14_11-12-23.png)

Step 3: Connect to CPU

click on CPU → In the System window click on Attach , now the Mode will be automatically switch to Up and you can see running state in bottom right corner as marked in below picture

![](../../../_images/2.0%20SAIL%20-%20How%20to%20Flash%20and%20debug%20SAIL/image-2024-9-14_11-13-45.png)

Step 4: Load the elf

- Break the execution by clicking the Pause button in menu bar
- Now load the elf using the below command

Data.load.elf elf_file_path /Nocode

elf_file_name = ..\temp_j6e_mcu\Build\DASY_win_R52_mcu\DASY_win_R52_mcu.elf

Example: Data.load.elf C:\GIT\Sail_J6E_Base\temp_j6e_mcu\Build\DASY_win_R52_mcu\DASY_win_R52_mcu.elf /Nocode

Now we can see the execution will be the while loop. Change PC to main function and then we can see OS tasks will be scheduled.

![](../../../_images/2.0%20SAIL%20-%20How%20to%20Flash%20and%20debug%20SAIL/image-2024-9-29_13-45-47.png)

To reset and debug again → just follow step 4 after power reset
