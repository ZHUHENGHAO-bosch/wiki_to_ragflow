## Overview

SAIL domain has Boot configuration and this page is to analyze what is boot configuration and howe to use this in SAIL platform

## SAIL Boot configuration

Same like Option bytes in Renesas and BMHD (Boot Mode header) in Tricore. But QC BootCfg is controlled by GPIOs not via a Flash memory

Boot config can control

- WDG Disable
- SAIL boot type
- BIST
- RGMII voltage selection
- Cluster and Core config
- Forse USB

Important Notes:

Ensure that there are no unintentional external pulls on these SAIL_IOs. The external pulls can force the SA8775P device to enter the secure boot or select a different boot device

For initial bring-up purpose, ensure all SAILSS_BOOT_CONFIG pins are configured to the settings in red as default

SAIL_IO_37, SAIL_IO_39, SAIL_IO_43, SAIL_IO_45, and SAIL_IO_51 have internal default PD enabled on SA8775P ES1 samples and have internal default PU enabled on SA8775P ES2 and later samples.

| IO Number | Description | Function in Mission Mode | Description in XBL |
| --- | --- | --- | --- |
| SAIL_IO_5 | Fast Boot Mode entry | UART_RX in MD and SAIL domain | 0 –Fast Boot Mode entry 1 –Mission Mode entry |
| SAIL_IO_62 | Device Programmer Mode entry | General purpose sideband interrupt between SAIL domain and VIP | 0 –Mission Mode entry 1 –Device Programmer Mode entry |

Fast Mode and Device Programmer Mode are software features, For details, refer to QAM8775P IVI Boot and Corebsp Architecture Technical Overview

GPIO_35, SAIL_IO_5 and SAIL_IO_62 are used in Qualcomm reference platform by default. However, they are not hardware boot configuration pins, and the software functions can be reassigned to other GPIO or SAIL_IOs if required.

VIP needs to control the GPIO and SAIL_IOs into the right state based on the boot requirement during boot up. Both signals are sampled in the XBL stage
