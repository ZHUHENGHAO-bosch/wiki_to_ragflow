## Overview

This page is to summarizes the boot sequence of QAM 8225/8775

### Processors available in Chipset (8255)

| Subsystem | Executes in |
| --- | --- |
| Application processor subsystem (APSS) | APSS bootROM, IMEM, and LPDDR5 |
| Safety Island (SAIL) | SAIL boot NOR, SRAM |
| Always-on processor (AOP) | AOP code RAM |
| System hub resource manager (SHRM) | SHRM code RAM |
| SSC –Qualcomm®Hexagon™DSP | LPDDR5 |
| APPS CPUSS control processor (CPUCP) | IMEM, LPDDR5 |
| Video processing unit (VPU) | LPDDR5 |
| Low-power audio subsystem (LPASS) | LPDDR5 |
| Compute subsystem | LPDDR5 |
| Camera image signal processor (ISP) | LPDDR5 |
| Qualcomm®Neural Processing SDK (NPU) | LPDDR5 |

### Boot up sequence of SAIL and MD

1. After the chipset is reset, the SAIL and Qualcomm Kryo CPUGold core 0 comes out of reset

1a. SAIL configures the pre-built-in self-test (pre-BIST) and go into wait for interrupt (WFI) and hardware BIST controller unit (HBCU), which is a hardware controller performs the BIST on SAIL. SAIL PBL initializes hardware and loads SAIL hypervisor to run

1b. SAIL hypervisor loads the EL1 image

1c. APPS core0 executes application primary boot loader (APPS PBL) pre-BIST configuration and goes into WFI and wait for SAIL communication to run BIST on APSS

After Phase-1 BIST, SAIL HYP sends post BIST reset to APSS, APSS reset and the KryoGold core 0, APPS PBL initializes hardware (timer, PRNG, clocks, and so on), enables caches and memory management unit (MMU), and detects the boot device according to the boot option configuration

2a. APPS PBL loads and authenticates XBL loader(region #1) from the bootdevice to the Boot IMEM
